# 05-插件系统与工具调用

插件系统是 Cherry Studio AI 链路的扩展主轴。当前实现是“aiCore 插件生命周期 + 渲染层插件编排”双层模型。

## 双层结构

1. aiCore 插件引擎：`packages/aiCore/src/core/plugins/*`
2. 产品插件装配：`src/renderer/src/aiCore/plugins/PluginBuilder.ts`

## 插件生命周期

aiCore 插件支持以下核心钩子：

- `configureContext`
- `resolveModel`
- `transformParams`
- `transformStream`
- `transformResult`
- `onRequestStart`
- `onRequestEnd`
- `onError`

执行顺序受 `enforce` 影响：`pre -> normal -> post`。

## PluginBuilder 的编排策略

`PluginBuilder.ts` 会按运行条件拼装插件，例如：

- `createTelemetryPlugin`：开发模式下的追踪增强
- `createPdfCompatibilityPlugin`：不支持 PDF 的 provider 兼容处理
- `createReasoningExtractionPlugin`：推理内容提取
- `createSimulateStreamingPlugin`：非流式结果模拟流式输出
- `webSearchPlugin`：模型原生搜索能力
- `searchOrchestrationPlugin`：工具编排入口（知识库/记忆/网页）
- `createPromptToolUsePlugin`：不支持原生 function calling 时的 prompt 模拟工具调用

## 工具调用的两条路径

### 路径 A：原生 Function Calling

- 模型原生支持工具 schema 和 tool call 事件。
- `AiSdkToChunkAdapter` 解析工具调用事件并交给 `ToolCallChunkHandler`。
- UI 直接接收工具调用状态与结果。

### 路径 B：Prompt Tool Use 模拟

- 适用于不支持原生 function calling 的模型。
- 通过 `createPromptToolUsePlugin` 在提示词中注入工具调用协议。
- 解析模型输出中的工具调用意图，再执行 MCP 工具并回填结果。

两条路径在 UI 层最终统一成 `ChunkType` 事件，不影响消息渲染层。

## 搜索编排插件（关键）

`searchOrchestrationPlugin.ts` 负责三件事：

1. `onRequestStart`：意图识别，判断是否启用网页搜索/知识搜索/记忆搜索。
2. `transformParams`：动态注入对应工具（`KnowledgeSearchTool`、`MemorySearchTool`、`WebSearchTool`）。
3. `onRequestEnd`：对话结束后触发记忆写回流程（后台异步处理）。

这使搜索与记忆能力从“固定开关”升级为“按轮次动态决策”。

## 与 StreamProcessingService 的衔接

工具调用事件统一进入 `StreamProcessingService`，再映射到回调：

- `onToolCallPending`
- `onToolCallInProgress`
- `onToolCallComplete`
- `onToolArgumentStreaming`
- `onExternalToolInProgress`
- `onExternalToolComplete`

这保证 UI 层无需感知底层工具协议差异。
