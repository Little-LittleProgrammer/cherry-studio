# 10-关键类型与扩展指南

本章给出扩展 AI 能力时的落点建议，避免改错层级。

## 常见扩展需求与改动位置

### 1. 新增模型 Provider

优先改动：

- `src/renderer/src/aiCore/provider/config/*`
- `src/renderer/src/aiCore/provider/factory.ts`
- `packages/aiCore/src/core/providers/*`（如需内核级注册能力）

不要直接在页面组件里拼 Provider 逻辑。

若新增 provider 还包含配套产品能力，通常还要同步检查：

- `src/renderer/src/services/ApiService.ts` 中的模型拉取链路
- Provider OAuth / 自定义认证设置页
- 模型健康检查、模型管理、默认标签能力

### 2. 新增工具调用能力

优先改动：

- `src/renderer/src/aiCore/plugins/PluginBuilder.ts`
- `src/renderer/src/aiCore/plugins/searchOrchestrationPlugin.ts`
- `src/renderer/src/aiCore/tools/*`
- `src/main/services/MCPService.ts`（若需要主进程能力）

### 3. 新增知识检索策略

优先改动：

- `src/main/services/KnowledgeService.ts`
- `src/renderer/src/services/KnowledgeService.ts`
- `searchOrchestrationPlugin` 工具注入逻辑

避免把 RAG 检索逻辑下沉到 UI 页面。

若涉及文档导入质量或召回精度，也常常需要同步改：

- `src/main/knowledge/preprocess/*`
- `src/main/knowledge/reranker/*`

### 4. 新增记忆策略

优先改动：

- `src/main/services/memory/MemoryService.ts`
- `src/renderer/src/services/MemoryService.ts`
- `src/renderer/src/services/MemoryProcessor.ts`（若策略在渲染侧触发）

### 5. 扩展 Agent/Claude Code

优先改动：

- `src/main/services/agents/services/*`
- `src/main/services/agents/services/claudecode/*`
- 渲染侧 `messageThunk` Agent 分支

注意 Agent 子系统当前迁移状态，避免做数据模型扩张。

### 6. 新增 Web Search Provider

优先改动：

- `src/renderer/src/providers/WebSearchProvider/*`
- `src/renderer/src/config/webSearchProviders.ts`
- `src/renderer/src/services/WebSearchService.ts`
- `searchOrchestrationPlugin` 或 `webSearchPlugin` 的接入策略

注意区分：

- 模型原生 web search
- 外部 web search provider

两者不是同一条扩展链路。

### 7. 新增知识预处理 Provider

优先改动：

- `src/main/knowledge/preprocess/PreprocessProviderFactory.ts`
- `src/main/knowledge/preprocess/*`
- 渲染侧知识库设置页与 preprocess provider 配置

### 8. 新增 Reranker Strategy

优先改动：

- `src/main/knowledge/reranker/strategies/*`
- `src/main/knowledge/reranker/StrategyFactory.ts`

## 关键类型与对象

建议重点阅读：

- `AiPlugin`、`AiRequestContext`（`packages/aiCore/src/core/plugins/types.ts`）
- `RuntimeExecutor`（`packages/aiCore/src/core/runtime/executor.ts`）
- `StreamTextParams`（渲染侧 `types/aiCoreTypes`）
- `Chunk` / `ChunkType`（`src/renderer/src/types/chunk.ts`）
- `MCPTool`、`MCPServer`（共享类型）
- Agent 会话相关类型（`@types` 下 Agent 定义）

补充建议阅读：

- Provider / model 能力标签相关类型（vision、tool use、web search、embedding、rerank）
- preprocess provider 与 knowledge base 配置类型
- memory config / memory search 相关类型

## 变更守则

1. 执行能力放 `ai-core`，产品策略放渲染层编排。
2. 需要系统资源或持久化的能力放主进程服务。
3. UI 只消费统一 Chunk 与状态，不耦合底层协议细节。
4. 日志统一走 `loggerService`，不要使用 `console.log`。
5. 大改前先确认是否命中 v2 refactor 限制文件。

## 交付前检查

按仓库约定，文档重构之外的代码变更应执行：

- `pnpm lint`
- `pnpm test`
- `pnpm format`

若只是本目录文档改动，可保持“无代码行为变化”并确保链接与路径正确。
