# 07-知识库与长期记忆系统

知识库与记忆都属于“增强上下文”能力，但它们的目标和实现不同：

- 知识库：面向外部文档检索（RAG）
- 记忆：面向用户长期偏好与事实沉淀

## 知识库链路（RAG）

主进程入口：`src/main/services/KnowledgeService.ts`

核心实现要点：

1. 使用 `@cherrystudio/embedjs` 构建 RAG 应用。
2. 存储层基于 `LibSqlDb`，每个知识库独立路径。
3. 支持多源数据：文件、目录、网页、站点地图、笔记。
4. 通过 Embedding + 相似度搜索返回候选片段。
5. 可选 Reranker 做二次排序。

渲染侧接入：

- 通过 `window.api.knowledgeBase.*` IPC 调用主进程服务。
- 在消息请求前，按策略注入知识搜索 prompt 或工具调用。

## 记忆链路（Memory）

主进程入口：`src/main/services/memory/MemoryService.ts`

核心能力：

1. 本地 `libsql` 存储记忆与历史。
2. 记忆写入前做去重（hash + 相似度阈值）。
3. Embedding 向量检索（hybrid search）。
4. 维护记忆历史（ADD/UPDATE/DELETE 等语义）。

渲染侧入口：`src/renderer/src/services/MemoryService.ts`

职责：

- 维护当前用户上下文
- 调主进程 API 完成增删改查
- 同步记忆配置（embedding 模型、维度等）

## 搜索与记忆如何参与对话

`searchOrchestrationPlugin` 在一次请求中可触发：

1. 意图分析：判断是否需要知识/网页/记忆检索。
2. 动态注入工具：知识搜索、记忆搜索、网页搜索。
3. 对话结束后记忆写回：抽取事实并异步入库。

这样可以把“检索”与“写回”放入同一生命周期，而不是散落在页面逻辑。

## 设计边界

- 知识库关注“文档事实检索”，不存用户长期行为偏好。
- 记忆关注“用户长期状态”，不承担通用文档 RAG。
- 两者都由主进程持久化，渲染层只编排与展示。
