# 知识库与长期记忆全景流程

本文档详细介绍 Cherry Studio 中知识库（Knowledge Base）与长期记忆（Long-term Memory）的完整实现流程。

## 一、整体架构概览

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                              用户输入                                        │
│                         (聊天消息 / 助手配置)                                 │
└─────────────────────────────────────────────────────────────────────────────┘
                                       │
                                       ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│                    searchOrchestrationPlugin                                │
│                         (搜索编排插件)                                       │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │ onRequestStart: 意图识别 (LLM 调用 #1)                                │   │
│  │   • 分析用户消息，判断需要哪些搜索                                      │   │
│  │   • 提取搜索关键词                                                    │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │ transformParams: 工具注入                                             │   │
│  │   • 注入知识库搜索工具 (builtin_knowledge_search)                    │   │
│  │   • 注入记忆搜索工具 (builtin_memory_search)                         │   │
│  │   • 注入网页搜索工具 (builtin_web_search)                            │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │ onRequestEnd: 记忆存储 (LLM 调用 #4)                                 │   │
│  │   • 对话结束后自动提取并存储记忆                                       │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────────────────────┘
```

---

## 二、大模型调用总览

在整个知识库和记忆流程中，共涉及 **4 次大模型调用**：

| 序号 | 调用时机 | 用途 | 模型类型 |
|------|----------|------|----------|
| #1 | 意图识别阶段 | 分析用户消息，提取搜索关键词 | 主模型 (GPT-4o 等) |
| #2 | 知识库搜索时 | 将查询文本向量化 | Embedding 模型 |
| #3 | 记忆搜索时 | 将查询文本向量化 | Embedding 模型 |
| #4 | 记忆存储时 | 提取事实 & 更新记忆 | 主模型 (GPT-4o 等) |

## 四、知识库 Embedding 调用详解

### 4.1 Embedding 模型配置

```typescript
// 知识库配置
interface KnowledgeBaseParams {
  embedApiClient: {
    model: string        // 如: "text-embedding-3-small"
    provider: string     // 如: "openai", "ollama", "voyageai"
    apiKey: string
    baseURL: string
  }
  dimensions: number      // 向量维度 (如: 1536, 1024, 256)
}
```

### 4.2 支持的 Embedding 提供商

| 提供商 | 模型示例 | 向量维度 |
|--------|----------|----------|
| OpenAI | text-embedding-3-small, text-embedding-3-large | 1536, 3072 |
| Azure OpenAI | text-embedding-ada-002 | 1536 |
| Ollama | llama2, mxbai-embed-large | 768, 1536 |
| VoyageAI | voyage-large-2, voyage-code-2 | 1024, 1536 |

### 4.3 Embedding 调用时机

```
文档入库阶段:
  文档 → 分块 → embedDocuments() → 向量存储 → LibSQL

搜索阶段:
  查询 → embedQuery() → 向量 → 相似度搜索 → 结果

记忆存储阶段:
  新事实 → embedQuery() → 向量 → 相似度检查 (>0.85 跳过) → 存储

记忆搜索阶段:
  查询 → embedQuery() → 向量 → 相似度搜索 → 结果
```

### 5.3 解析流程

#### 5.3.1 解析代码

使用 `fast-xml-parser` 库进行解析：

```typescript
// src/renderer/src/utils/extract.ts
import { XMLParser } from 'fast-xml-parser'

export const extractInfoFromXML = (text: string): ExtractResults => {
  const parser = new XMLParser({
    isArray: (name) => {
      return name === 'question' || name === 'links'
    }
  })
  return parser.parse(text)
}
```

```typescript
const chatHistory = lastAnswer ? `assistant: ${getMessageContent(lastAnswer)}` : ''

// 获取用户问题
const question = "帮我查一下 Python 3.12 的新特性，同时看看我之前整理的 Python 学习资料里有没有相关内容"

// 格式化提示词
const formattedPrompt = SEARCH_SUMMARY_PROMPT
  .replace('{chat_history}', chatHistory)
  .replace('{question}', question)
```

#### 步骤 2: 调用 LLM 生成意图分析

```typescript
// 使用 LLM 分析意图
const { text: result } = await generateText({
  model: context.model as LanguageModel,
  prompt: formattedPrompt
})
```

#### 步骤 3: 解析 LLM 返回

```typescript
// 解析 XML 结果
const parsedResult = extractInfoFromXML(result)
```

#### 步骤 4: 存储分析结果供后续使用

```typescript
// 存储到 context 中
intentAnalysisResults[context.requestId] = parsedResult
```

#### 步骤 5: transformParams 阶段使用

```typescript
// 根据分析结果注入工具

// 1. 网页搜索工具
if (parsedResult.websearch) {
  params.tools['builtin_web_search'] = webSearchToolWithPreExtractedKeywords(
    assistant.webSearchProviderId,
    parsedResult.websearch,
    context.requestId
  )
}

// 2. 知识库搜索工具
if (parsedResult.knowledge) {
  params.tools['builtin_knowledge_search'] = knowledgeSearchTool(
    assistant,
    parsedResult.knowledge,
    getMessageContent(userMessage),
    topicId
  )
}
```
